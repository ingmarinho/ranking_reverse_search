Project: local reverse-image sourcing tool for video clips
Goal: A locally-run NiceGUI web app. User pastes a video URL (compilation/ranking video), the app downloads it, detects scenes, lets the user pick representative frames per scene, gives them buttons to reverse-image-search those frames on external engines, and lets them paste back a found source URL to download the original clip. Single user, runs on localhost, no hosting.
Stack & dependencies:

Python 3.11+, NiceGUI as the only UI framework (it bundles its own FastAPI server — don't add a separate one).
yt-dlp (used as a library, from yt_dlp import YoutubeDL, never shelled out).
scenedetect (PySceneDetect) with ContentDetector.
opencv-python (PySceneDetect dependency).
ffmpeg as a system binary (assume it's on PATH; surface a clear error if missing).
SQLite via stdlib sqlite3 for job persistence.
use imgbb api for hosting images for reverse image searches.
